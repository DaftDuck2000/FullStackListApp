# BFF API Reference

Base URL: `http://localhost:3001`

The BFF (Backend-For-Frontend) is a thin Express proxy that forwards all requests to the FastAPI backend at `http://localhost:8000`. It adds no business logic — all filtering, validation, and data storage happens in the backend API.

All routes mirror the backend API with an `/api` prefix.

| Method | BFF Endpoint | Proxies To |
|---|---|---|
| `GET` | `/api/items` | `GET /items` |
| `POST` | `/api/items` | `POST /items` |
| `DELETE` | `/api/items` | `DELETE /items` |
| `POST` | `/api/reset` | `POST /reset` |

Query parameters for `GET /api/items` are forwarded as-is: `name`, `priority`, `method`.

Request bodies for `POST` and `DELETE` are forwarded verbatim.

> **Note:** In development, the frontend at `:5173` accesses the BFF through Vite's proxy configuration, so the frontend code uses `/api/items` directly without specifying `http://localhost:3001`.

## Proxy Behaviour

```javascript
// GET /api/items?priority=High
// Forwards to: GET http://localhost:8000/items?priority=High
// Returns: response from backend verbatim

// POST /api/items  { "name": "...", "priority": "...", "method": "..." }
// Forwards to: POST http://localhost:8000/items
// Returns: response from backend verbatim
```

## Example with `curl`

```bash
# List all items
curl http://localhost:3001/api/items

# List with filter
curl "http://localhost:3001/api/items?priority=High&method=1"

# Add an item (id is auto-generated by the backend)
curl -X POST http://localhost:3001/api/items \
  -H "Content-Type: application/json" \
  -d '{"name":"Test","priority":"Medium","method":"2"}'

# Delete an item
curl -X DELETE http://localhost:3001/api/items \
  -H "Content-Type: application/json" \
  -d '{"id":99}'

# Reset
curl -X POST http://localhost:3001/api/reset
```
